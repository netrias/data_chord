V1 variations — non-color selection cues

Building on V1 (tinted-active chips), but layering shape, weight, depth, and iconography so the active state survives in monochrome. Each variant lists the additional cues it stacks on top of color tinting. Use the Grayscale toggle above to validate.

Single-cue variations (each adds one non-color affordance)

V1a · Weight + size bump

subtle
tint font-weight 500 → 700 border 1px → 2px slightly more padding

Active chip thickens — heavier border, bolder text, a touch more breathing room. Survives grayscale via stroke weight.

Lightest-touch option. May still feel ambiguous next to a hover state — use only if you want the bar to stay quiet.

V1b · Leading checkmark

explicit
tint leading ✓ glyph font-weight 600

Active chip prepends a checkmark. Borrows the universal "selected" affordance from checkboxes — unambiguous and screen-reader friendly when paired with aria-pressed.

Width shifts when toggling — chips reflow slightly. Worth it for clarity, but worth noting.

V1c · Bottom underline bar

tab-like
tint 3px underline below chip font-weight 600

Active chip grows a colored underline. Borrows the tab affordance without flattening the chip into a tab.

Reads as both chip and tab. Good middle ground.

V1d · Inset / pressed

tactile
tint inset shadow (depressed) translateY(+1px) font-weight 600

Active chip looks pushed in — like a toggle button held down. Strong tactile metaphor; the depth difference is obvious in grayscale.

Pairs nicely with the existing tactile button language elsewhere in the app.

V1e · Outer ring

focus-style
tint 2px offset ring font-weight 600

Active chip wears a focus-ring style outer halo. Distinct shape difference — impossible to miss in grayscale.

May visually conflict with keyboard focus rings — needs a distinct focus style if adopted.

V1f · Lift (raised)

elevated
tint translateY(-2px) drop shadow font-weight 600

Active chip floats above the others. Inverse of V1d — selection is "raised" rather than "pressed in." Subtle but noticeable in grayscale.

Pairs with hover-lift conventions used elsewhere; risks looking like a hover state.

V1g · Downward caret

spatial cue
tint caret pointing at the table font-weight 600

Active chip sprouts a small downward triangle pointing at the filtered list. Literally says "this filter shows what's below."

Most semantic but most novel — risks reading like a dropdown indicator.

V1h · Indicator dot (radio)

familiar mental model
tint leading dot fills when active font-weight 600

Every chip has a small leading circle — hollow when inactive, filled when active. Mirrors radio-button semantics, which matches the single-select behavior.

Strongest accessibility story — the dot is meaningful even with no color and no width change. Slightly more visual noise.

Strongest combination

V1·★ · Kitchen sink (recommended for accessibility)

redundant cues
tint leading ✓ font-weight 700 outer ring

Stacks weight, glyph, and ring on top of the color tint. Active state is unmissable in any color condition. Slightly louder than the singles — appropriate when filters drive the workflow.
